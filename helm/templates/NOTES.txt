NEXT STEPS
──────────

1. Build and push the custom image:
   cd {{ .Chart.Name }}/docker
   REGISTRY=ghcr.io/yourusername TAG={{ .Chart.AppVersion }} ./build.sh

2. Add your TLS domains to values.yaml:
   tls.certManagerCSI.dnsNames:
     - "*.yourdomain.com"
     - "yourdomain.com"

3. Add your L4 (TCP/UDP) routes to values.yaml:
   l4.hostPorts:
     - port: 53
       protocol: UDP
   l4.routes:
     - address: ":53"
       upstream: "udp/kube-dns.kube-system.svc.cluster.local:53"

   Then run: helm upgrade {{ .Release.Name }} ./helm

4. Add HTTP/HTTPS routes by editing the routes ConfigMap directly:
   kubectl edit configmap {{ include "caddy.fullname" . }}-routes -n {{ .Release.Namespace }}

   Each .caddy key becomes an imported Caddyfile snippet. Example:
     blog.caddy: |
       blog.yourdomain.com {
         tls /certs/tls.crt /certs/tls.key
         import security
         import upstream ghost.blog.svc.cluster.local:2368
       }

   Stakater Reloader will restart pods automatically when the ConfigMap changes.
